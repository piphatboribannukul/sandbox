🗺️ FRC Contour Model

      วิธีคำนวณ: Zone nearest-neighbor จาก สจ.สูบจ่าย/โรงงานผลิต

      ตัด TR1/TR2/TR3/MTR ออก (อุโมงค์ส่งน้ำ ไม่จ่ายโดยตรง)

      decay: EPANET C = C₀·e-K·(L/v) | K = kb + kw×(4/D)

      

      k รายสถานี (STATION_K_OVERRIDE):

      fit จากข้อมูลจริง 3 เดือน (ม.ค.–มี.ค. 2569) 61 สถานี

      อัปเดต k: ส่ง xlsx ใหม่ → รัน fit_k.py → inject HTML

⚡ EC Contour Model

      วิธีคำนวณ (Hybrid IDW):

      · Zone anchor จาก สถานีสูบจ่าย/โรงงานผลิตน้ำที่ใกล้ที่สุด

      · Monitor sensors ในโซนเดียวกัน มีอิทธิพลด้วย IDW p=4 รัศมี ~17 km

      · Monitor ต่างโซน weight ลดลง 10× ป้องกัน contour ข้ามโซน

      กราฟคาดการณ์ EC:
      · เส้นทึบ = ค่าจริงย้อนหลัง (localStorage เก็บทุก 4 นาที)

      · เส้นปะ = blend ec_ปัจจุบัน → ec_ต้นทาง ตาม travel time

      · สถานีผสมน้ำหลายแหล่ง → weighted average ก่อน blend

      ข้อจำกัด:
      · EC history เพิ่งเริ่มเก็บ → เส้นจริงในกราฟยังสั้น

      · IDW ไม่รู้จักโครงสร้างท่อจริง → contour รอยต่อโซนอาจไม่แม่น

      · ค่า EC จาก API บางสถานีอาจมี delay หรือ missing
